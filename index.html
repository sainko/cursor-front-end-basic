Перелік основних команд

git config --global user.name sainko (im`я користувача) git config --global user.email saienko.al@gmail.com (e-mail користувача) git init - створення нового Git репозитарію git status - відображає стан робочого каталогу та розділу проіндексонваних файлів git add - додає зміни з робочого каталогу в розділ проіндексованих файлів git commit - робить для проекту знімок поточного стану змін, доданих до розділу проіндексованих файлів git log - перегляд історії комітів (лог проэкту) git checkout - переміщення між гілками проєкту (зміна посилання в покажчику HEAD) git branch - это команда для управления ветками в репозитории Git git push - використовується для вивантаження вмісту локального репозиторію у віддалений репозиторій git pull - спочатку забирає зміни із зазначеного віддаленого репозиторію, а потім намагається злити їх із поточною гілкою git merge - злиття окремих частин проєкту у загальну історію git clone - створює дублікат репозитарію .gitignore - файл, що вміщує список файлів, які потрібно ігнорувати при операціях

Advanced

git revert - зберігає коміти, що відміняються, і створює новий коміт з операцією, зворотною останньому комміту git reset - скасування локальних змін у репозиторії Git git rebase - заміна одного комміту в основі гілки на інший git cherry-pick - бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміту в поточній гілці git stash - дозволяє на якийсь час відкласти зміни, зроблені в робочій копії, щоб ви могли застосувати їх пізніше
